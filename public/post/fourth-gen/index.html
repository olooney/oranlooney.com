The dream of fourth generation programming languages was to have a programming language that felt more like natural language and were thus simultaneously powerful and flexible than third generation languages, while being so easy to use that even non-programmers could use them. In brief, the computer on Star Trek.

fourth_gen is a Python package that makes that dream a reality. Now, thanks to the power of LLMs, you can specify function behavior in plain English, without ever having to write a single line of code. Not only that, but the generated code is capable of handling novel or unexpected cases, by delegating to LLMs as needed to cover cases which are too vague or require too much judgement to implement as a rigid algorithm.

This paradigm allows the rapid development of powerful programs that combine the performance of traditional computing paradigms with the flexibility of LLMs.

Conclusion

This is not a serious library; in fact I can pretty definitively say that it’s a Bad Idea™. Too magical, difficult to inspect or code review, and unstable between runs. Another problem is that decomposing an application into functions and providing the exact type annotations for all inputs and outputs is quite burdensome - that part of the problem is often more difficult than merely fleshing out the implementation of the functions. Someone who doesn’t want to write Python code isn’t going to want to do that either.

In that regard, the usual approach using a tool like GitHub Copilot or simply asking ChatGPT for code is generally superior this approach.

Still, it’s fun to imagine a day when we can simply talk to our computers in plain English.
